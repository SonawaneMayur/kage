# Slide 1 — Title (target: 30s)

## Hook line
> "Today's pipelines emit GBs of logs about CPU and memory — and almost nothing about the data flowing through them. KAGE fixes that in four lines of Python."

## Key talking points
1. KAGE = Universal Pipeline Observability — emphasis on **universal**.
2. Works across PySpark, Spark SQL, Databricks, DLT/Lakeflow, dbt, Airflow.
3. Drop-in: import + log, no platform lock-in, no agent to deploy.
4. Already installable from PyPI: `pip install kage`.

## Delivery notes
- Smile, slow down on "four lines."
- Don't read bullets — they're the audience's, not yours.
- Transition: *"So why do we need this in the first place?"* → next slide.
