Event Flow — One Decorator, Many Hops

Trace of @kage_dlt_table firing
  1. DLT runtime invokes the wrapped function.
  2. KAGE wrapper runs the user body and captures return value.
  3. Exception path → emit dataset_event with status=FAILED, error_type, stack_trace; re-raise.
  4. Success path → infer record_count (skipped by default for DLT; explicit via record_count_fn).
  5. KageLogger._safe_emit merges context + custom_fields under the lock.
  6. MultiTransport writes JSONL to event_type=dataset_event/dt=…/part-*.jsonl.
  7. DLT registers the returned DataFrame as a managed table.
  8. Analytics queries read the partitioned JSONL directly (no ingest step).
Streaming-safe by design

_infer_record_count checks isStreaming first and short-circuits to 0 — so the wrapper never calls .count() on a streaming DataFrame (which would block forever).
