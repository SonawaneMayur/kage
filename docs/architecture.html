Two Paradigms, One Stream

ETL trace — @kage_dlt_table
  1. DLT runtime invokes the wrapped function.
  2. KAGE wrapper runs the body, captures return value.
  3. On raise → dataset_event with status=FAILED + stack_trace; re-raise.
  4. Success → infer record_count (skipped by default; opt-in via record_count_fn).
  5. _safe_emit writes JSONL to event_type=dataset_event/dt=…/part-*.jsonl.
  6. Analytics SQL reads the partitions directly — no ingest.
Agentic trace — @agent + @step + @tool
  1. @agent sets _active_job_id contextvar; emits job_run RUNNING.
  2. Root span pushed onto _active_span_stack (contextvar).
  3. Nested @step / @tool / @llm_call read parent from the stack → emit with parent_span_id.
  4. Parallel children (asyncio.gather / ThreadPool with copy_context) inherit the same parent.
  5. log_llm_usage(...) attaches prompt_tokens · completion_tokens · cost_usd · model to the active span.
  6. Span end → task_run with latency_ms, status SUCCESS / FAILED.
  7. Same JSONL store as ETL — analytics queries rebuild the call tree via parent_span_id.
Streaming-safe (ETL)

_infer_record_count checks isStreaming first and short-circuits to 0 — never calls .count() on a streaming DataFrame.

Concurrency-safe (Agentic)

contextvars propagate across await boundaries automatically. ThreadPool needs contextvars.copy_context().run(fn) per submission — one fresh copy per worker because a Context can only be entered once at a time.
